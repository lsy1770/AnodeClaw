# 自动化配方指南

## 创建 Cron 定时任务
使用 `cron_manage` 工具创建定时任务：

```
调用 cron_manage:
  action: "create"
  name: "任务名称"
  schedule: "cron表达式"
  toolName: "要执行的工具名"
  toolInput: { ... 工具参数 ... }
```

## Cron 表达式参考
| 表达式 | 含义 |
|--------|------|
| `*/5 * * * *` | 每5分钟 |
| `0 * * * *` | 每小时 |
| `0 9 * * *` | 每天9点 |
| `0 9 * * 1-5` | 工作日9点 |
| `0 */2 * * *` | 每2小时 |
| `0 9,18 * * *` | 每天9点和18点 |

## 管理定时任务
- 查看所有任务: `cron_manage` action: "list"
- 暂停任务: `cron_manage` action: "pause", name: "任务名"
- 恢复任务: `cron_manage` action: "resume", name: "任务名"
- 删除任务: `cron_manage` action: "delete", name: "任务名"

## 批量操作模式
对于需要对多个目标执行相同操作的场景：
1. 准备目标列表
2. 使用 `code_exec_async` 编写循环逻辑
3. 每次操作间加入适当延迟（避免过快操作）
4. 记录每步结果用于后续分析

## 自动化最佳实践
- 每个自动化步骤后验证结果
- 设置合理的超时和重试机制
- 记录执行日志便于排查问题
- 避免过于频繁的定时任务（考虑电池消耗）
- 关键操作前截图保存现场
